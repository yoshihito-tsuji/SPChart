# 三者間協働開発ガイド

## 概要

SPChartプロジェクトは、3者による協働開発モデルを採用しています。

## 役割定義

### Yoshihitoさん（プロジェクトオーナー）

- **責務**: 最終意思決定、企画・要件定義、プロジェクト管理
- **権限**: すべての決定に対する最終承認権
- **コミュニケーション**: 日本語で直接指示

### Codex（アーキテクト）

- **責務**: 設計・アーキテクチャ策定、技術方針決定、設計レビュー
- **権限**: 技術的な設計判断
- **コミュニケーション**: From/To形式でClaude Codeに指示

### Claude Code（実装エンジニア）

- **責務**: コード実装、品質管理、テスト、技術課題の報告
- **権限**: 実装レベルの判断
- **コミュニケーション**: From/To形式で報告・質問

## 意思決定優先順位

```
1. Yoshihitoさんの意図・要望 【最優先】
        ↓
2. Codexのアーキテクチャ・設計方針
        ↓
3. 技術的なベストプラクティス
```

## コミュニケーション形式

### AI同士の通信

```markdown
From: [送信者]
To: [受信者]

[本文]
```

### 例

```markdown
From: Claude Code
To: Codex

注意係数の計算において、分母が0になるケースの処理について確認させてください。
現在の仕様では...
```

## セッション開始手順

1. `README.md` を確認
2. `CLAUDE.md` または `Codex.md` を確認
3. 最新の `LOG/YYYY-MM-DD.md` を確認
4. `DECISIONS.md` で確定事項を確認
5. 作業開始、ログに記録

## ログ管理

- **ファイル**: `LOG/YYYY-MM-DD.md`
- **番号形式**: `LOG_00001`（5桁ゼロパディング）
- **セクション**: `[PROPOSAL]`, `[REVIEW]`, `[PLAN]`, `[RUNLOG]`, `[DECISION]`

## 決定事項の記録

重要な決定は `DECISIONS.md` に1行形式で記録：

```
[YYYY-MM-DD] [カテゴリ] 決定内容 (決定者)
```
